# Customer Shopping Behavior Analysis

An end-to-end data analytics project covering data cleaning (Python), business-question analysis (SQL), and an interactive Power BI dashboard — built on a retail customer transactions dataset of ~3,900 records.

## Project Overview

This project analyzes customer shopping behavior across demographics, product categories, payment preferences, and subscription status to uncover patterns that can inform retail marketing and retention strategy. It demonstrates a full analytics workflow: raw data → cleaned dataset → SQL business insights → interactive BI dashboard.

## Tools Used

- **Python (Pandas)** — data cleaning and feature engineering
- **MySQL** — business-question querying (CTEs, window functions, aggregations)
- **Power BI** — interactive dashboard and visualization
- **Jupyter Notebook** — analysis documentation

## Dataset

`data/customer_shopping_behavior.csv` contains 3,900 customer transaction records with 18 attributes, including:

- Demographics: Age, Gender, Location
- Purchase details: Item Purchased, Category, Purchase Amount, Review Rating
- Behavior: Subscription Status, Discount Applied, Previous Purchases, Frequency of Purchases
- Fulfillment: Shipping Type, Payment Method

## 1. Data Cleaning & Feature Engineering (Python)

Using Pandas, the raw dataset was cleaned and enriched:

- Imputed missing `Review Rating` values using the median rating per product category
- Standardized column names to snake_case for consistency
- Engineered `age_group` (Young Adult / Adult / Middle-aged / Senior) via quartile binning
- Engineered `purchase_frequency_days` by mapping purchase frequency labels (e.g. Weekly, Monthly, Annually) to numeric day intervals
- Verified `Discount Applied` and `Promo Code Used` were fully redundant, then dropped the duplicate column

## 2. SQL Business Analysis (MySQL)

Ten business questions were answered using MySQL, including window functions and CTEs:

| # | Business Question |
|---|---|
| 1 | Total revenue generated by male vs. female customers |
| 2 | Customers who used a discount but still spent above the average purchase amount |
| 3 | Top 5 products by average review rating |
| 4 | Average purchase amount: Standard vs. Express shipping |
| 5 | Subscriber vs. non-subscriber spend comparison |
| 6 | Top 5 products by discount usage rate |
| 7 | Customer segmentation (New / Returning / Loyal) by purchase history |
| 8 | Top 3 best-selling products within each category (window functions) |
| 9 | Subscription likelihood among repeat buyers (5+ previous purchases) |
| 10 | Revenue contribution by age group |

Full queries: [`sql/customer_behaviour_analysis_queries.sql`](sql/customer_behaviour_analysis_queries.sql)

## 3. Power BI Dashboard

An interactive dashboard (`powerbi/customer_behavior_dashboard.pbix`) was built with slicers for Subscription Status, Gender, and Category, featuring:

- KPI cards for customer count, average purchase amount, and average review rating
- Revenue and sales volume by product category
- Subscription status split
- Customer distribution by payment method
- Sales by age group and gender

## Key Insights

- **Male customers generated significantly more revenue** than female customers ($157,890 vs. $75,191), despite the dataset skewing male in volume — a signal worth validating against customer counts before drawing marketing conclusions.
- **Clothing is the top-performing category**, contributing over $104K in revenue, more than double the next closest category (Accessories).
- **Only 27% of customers are active subscribers**, pointing to a large untapped opportunity for subscription-driven retention campaigns.
- **Average review rating sits at 3.75/5** across all products, with accessories like Gloves, Sandals, and Boots rating highest — useful for merchandising and upsell prioritization.
- **Average purchase amount is ~$59.76**, providing a baseline to measure the impact of future discount or loyalty initiatives.

## How to Reproduce

1. Load `data/customer_shopping_behavior.csv` into MySQL and run `sql/customer_behaviour_analysis_queries.sql`
2. Open `notebook/Customer_shopping_behaviour_analysis.ipynb` in Jupyter to review the cleaning/feature engineering steps
3. Open `powerbi/customer_behavior_dashboard.pbix` in Power BI Desktop to explore the interactive dashboard
